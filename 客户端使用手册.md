# WakeFusion 客户端使用手册

## 一、环境搭建

### 1.1 Conda 环境配置

系统需要两个独立的 conda 环境，Python 版本和依赖不同：

#### 环境1：`wakefusion`（主环境）
- **Python版本**：3.10+
- **用途**：音频服务、核心服务、Mock LLM Agent
- **安装依赖**：
  ```bash
  conda activate wakefusion
  pip install -r requirements.txt
  ```
- **主要依赖**：PyTorch (CPU版)、Silero VAD、NeMo、ZMQ、WebSockets

#### 环境2：`wakefusion_vision`（视觉环境）
- **Python版本**：3.10+
- **用途**：视觉服务（人脸检测、唇动检测、距离估计）
- **安装依赖**：
  ```bash
  conda activate wakefusion_vision
  pip install mediapipe opencv-python pyorbbecsdk numpy
  ```
- **主要依赖**：MediaPipe、OpenCV、Orbbec SDK

### 1.2 环境创建（首次使用）

```bash
# 创建主环境
conda create -n wakefusion python=3.10 -y
conda activate wakefusion
pip install -r requirements.txt

# 创建视觉环境
conda create -n wakefusion_vision python=3.10 -y
conda activate wakefusion_vision
pip install mediapipe opencv-python pyorbbecsdk numpy
```

## 二、启动方式

### 2.1 一键启动（推荐）

使用 PowerShell 启动脚本：

```powershell
.\launcher.ps1
# 选择 1：启动所有服务
# 选择 2：停止所有服务
```

**启动顺序**：
1. Vision Service（`wakefusion_vision` 环境）
2. Audio Service（`wakefusion` 环境）
3. Core Server（`wakefusion` 环境）

### 2.2 手动启动（调试用）

**终端1 - 视觉服务**：
```bash
conda activate wakefusion_vision
python -m wakefusion.services.vision_service --fps 15
```

**终端2 - 音频服务**：
```bash
conda activate wakefusion
python -m wakefusion.services.audio_service
```

**终端3 - 核心服务**：
```bash
conda activate wakefusion
python -m wakefusion.services.core_server
```

### 2.3 Web 页面启动

```bash
cd wakefusion_web
npm install  # 首次运行
npm run dev  # 开发模式（http://localhost:3000）
```

**注意**：Web 页面通过 WebSocket 连接到 Core Server（`ws://127.0.0.1:8765`）

### 2.4 Mock LLM Agent 启动（测试用）

当没有接入真实服务端时，使用 Mock 服务进行测试：

```bash
conda activate wakefusion
python tests/mock_llm_agent_simple.py
```

**功能**：
- 模拟 ASR 识别结果
- 模拟 AI 回复文本
- 模拟 TTS 音频流
- 支持 interrupt/stop 指令

## 三、后端主要模块

### 3.1 Vision Service（视觉服务）
- **功能**：人脸检测、唇动检测、距离估计
- **输出**：通过 ZMQ PUB 发布视觉数据（端口 5555）
- **环境**：`wakefusion_vision`

### 3.2 Audio Service（音频服务）
- **功能**：音频采集、VAD、KWS、音频播放
- **通信**：ZMQ PUB（5556）、ZMQ REP（5557）
- **环境**：`wakefusion`

### 3.3 Core Server（核心服务）
- **功能**：多模态融合、状态机管理、WebSocket 客户端
- **职责**：
  - 订阅 Vision/Audio 数据
  - 决策唤醒/截断逻辑
  - 连接 LLM Agent（WebSocket 客户端）
  - 转发字幕给 Web UI（WebSocket 服务端）
- **环境**：`wakefusion`

## 四、通信机制

### 4.1 后端内部通信（ZMQ）

| 模块 | 端口 | 协议 | 方向 |
|------|------|------|------|
| Vision → Core | 5555 | ZMQ PUB-SUB | 视觉数据流 |
| Audio → Core | 5556 | ZMQ PUB-SUB | 音频数据流 |
| Core → Audio | 5557 | ZMQ REQ-REP | 控制指令 |
| Core → Vision | 5564 | ZMQ PUB-SUB | 控制指令 |

### 4.2 后端与 Web UI 通信（WebSocket）

- **连接方式**：Core Server 作为 WebSocket 服务端
- **地址**：`ws://127.0.0.1:8765`
- **消息类型**：
  - `subtitle_user`：用户 ASR 字幕
  - `subtitle_ai`：AI 回复字幕
  - `subtitle_clear`：清空字幕
  - `playAudio`：Unity 音频播放指令

### 4.3 客户端与服务端通信（WebSocket）

- **连接方式**：Core Server 作为 WebSocket 客户端
- **地址**：`ws://127.0.0.1:8080/api/voice/ws?deviceId=xxx&token=xxx`
- **配置**：`config/config.yaml` → `llm_agent.host`
- **消息类型**：
  - **上行**：`audio_start`、`audio_end`、`audio_cancel`、音频二进制流
  - **下行**：`asr`、`text`、`route`、TTS 音频流、`interrupt`、`stop`

## 五、核心机制

### 5.1 多模态唤醒
- **纯语音唤醒**：KWS 置信度 ≥ 0.9
- **视觉降维打击**：4米内 + 正面率 ≥ 40% → 阈值降至 0.6

### 5.2 状态机
- **IDLE** → **VISUAL_WAKE** → **LISTENING** → **SPEAKING** → **LISTENING**
- 90秒全局超时、5秒宏观发呆、1.5秒微观截断

### 5.3 防抖机制
- **视觉进入防抖**：200ms
- **视觉离开防抖**：1000ms
- **物理防抖护盾**：TTS 结束后 0.8 秒内忽略音频

### 5.4 截断逻辑
- **微观截断**：有 trace_id 时，1.5秒静默
- **宏观发呆**：无 trace_id 时，5秒静默
- **连续6次发呆**：自动退出交互模式

## 六、注意事项

### 6.1 环境隔离
- 视觉服务必须在 `wakefusion_vision` 环境运行
- 音频和核心服务必须在 `wakefusion` 环境运行
- 不要混用环境，避免依赖冲突

### 6.2 启动顺序
- 先启动数据生产者（Vision/Audio）
- 再启动消费者（Core Server）
- Mock LLM Agent 可在 Core Server 之前或之后启动

### 6.3 端口占用
- 如果端口被占用，修改 `config/config.yaml` 中的端口配置
- 常用端口：5555（视觉）、5556-5557（音频）、8765（Web UI）、8080（LLM Agent）

### 6.4 配置文件
- 主配置文件：`config/config.yaml`
- 修改配置后需重启对应服务

### 6.5 日志级别
- 默认：INFO
- 调试：修改 `config/config.yaml` → `runtime.log_level: "DEBUG"`

### 6.6 硬件要求
- **麦克风**：XVF3800（推荐）或默认设备
- **摄像头**：Orbbec Femto Bolt（推荐）或 USB 摄像头
- **首次运行**：Silero VAD 模型会自动下载（需要网络）

### 6.7 测试建议
- 首次使用：先运行 `python tests/test_audio_driver` 验证硬件
- 完整测试：使用 Mock LLM Agent 进行端到端测试
- 生产环境：确保 LLM Agent 服务端已启动并配置正确
