## 📖 详细步骤

### 步骤 1：生成 TTS 正样本

**脚本**: `training/tts_positive_generator.py`
**功能**: 使用 10 种中文 TTS 声音 × 6 种语速风格，生成"你好小康"正样本
**输出**: `custom_dataset/xiaokang/` 目录下约 **180 条** TTS 正样本（全部 2 秒）

```powershell
conda activate wakefusion
python training/tts_positive_generator.py
```

**预计耗时**: 5-10 分钟（取决于网络速度）

> ⚠️ 需要联网，脚本会从微软 Edge-TTS 服务器下载语音。
> 内置了防限流和自动重试机制，遇到个别失败属于正常现象。

---

### 步骤 2：录制真人语音

**脚本**: `training/xvf3800_collector.py`
**功能**: 通过 XVF3800 麦克风阵列录制真人唤醒词和负样本

```powershell
conda activate wakefusion
python training/xvf3800_collector.py
```

**录制建议**:

| 类型 | 建议数量 | 说明 |
|------|---------|------|
| 你本人说"你好小康" | 20-30 条 | 不同距离、不同语速、不同音量 |
| 家人/同事说"你好小康" | 每人 10-15 条 | 🌟 核心泛化能力来源 |
| 真人负样本 | 20-30 条 | "小康小康"、"你好小刚"、咳嗽、日常对话等 |

> 💡 真人录音越多、声音越多样，模型泛化能力越强。
> 这是提升唤醒率的最关键步骤。

---

### 步骤 3：正样本数据增强

**脚本**: `training/positive_data_factory.py`
**功能**:
- 对 **真人录音** → 变调 + 变速 + 加噪（最大化裂变）
- 对 **TTS 样本** → 仅加噪 + 远场模拟（避免重复增强）
**输出**: 就地扩充 `custom_dataset/xiaokang/` 目录

```powershell
conda activate wakefusion
python training/positive_data_factory.py
```

**预计耗时**: 1-3 分钟

> 脚本会自动识别 `tts_` 前缀的文件并跳过变调变速，
> 只对真人录音做完整的数据增强流程。

---

### 步骤 4：生成 TTS 负样本

**脚本**: `training/tts_negative_generator.py`
**功能**: 使用 10 种声音 × 25 个易混淆短语 × 3 种风格，生成负样本
**输出**: `custom_dataset/others/` 目录下约 **750 条** TTS 负样本

```powershell
conda activate wakefusion
python training/tts_negative_generator.py
```

**预计耗时**: 15-30 分钟

---

### 步骤 5：(可选) 切割环境噪声

**脚本**: `training/negative_slicer.py`
**功能**: 将一段长环境录音（办公室、街道、咖啡厅等）切割成 2 秒片段
**输出**: `custom_dataset/others/` 目录

```powershell
# 1. 先将环境录音文件放到项目根目录，命名为 background_10min.wav
# 2. 运行切片脚本
conda activate wakefusion
python training/negative_slicer.py
```

> 💡 这一步是可选的，但强烈推荐。
> 真实环境噪声能显著降低误唤醒率。

---

### 步骤 6：分割训练集 / 验证集

**脚本**: `training/split_dataset.py`
**功能**: 自动扫描 `custom_dataset/` 下的所有数据，按组 80/20 分割
**输出**:
- `custom_dataset/manifests/train_manifest.json`
- `custom_dataset/manifests/val_manifest.json`

```powershell
conda activate wakefusion
python training/split_dataset.py
```

**核心规则**: 同一条原始录音的所有增强版本（变调/变速/加噪）会被分到同一个集合，
防止模型在验证集上"作弊"。

**预计耗时**: 几秒钟

> ⚠️ 每次新增数据后，都需要重新运行此脚本来更新清单。

---

### 步骤 7：训练模型 🔥

**脚本**: `fast_train.py`
**功能**: 从 MatchboxNet 预训练模型开始，用你的数据训练唤醒词模型
**输出**: `xiaokang_xvf3800_pro.nemo`

```powershell
conda activate wakefusion
python fast_train.py
```

**训练参数**:

| 参数 | 值 | 说明 |
|------|-----|------|
| 最大轮数 | 60 | 通常 15-30 轮会被 EarlyStopping 停止 |
| EarlyStopping | patience=10 | 连续 10 轮验证损失无改善则自动停止 |
| 批次大小 | 32 | |
| GPU | 1 块 | |

**预计耗时**: 10-30 分钟（取决于数据量和 GPU 性能）

**如何判断训练成功**:
- ✅ 训练损失 (train_loss) 持续下降
- ✅ 验证损失 (val_loss) 跟随下降后趋于稳定
- ✅ EarlyStopping 在合理的轮次（15-40 轮）自动停止
- ❌ 如果第 1 轮就停止了 → 数据可能有问题
- ❌ 如果跑满 60 轮都没停止 → 模型可能欠拟合，检查数据量是否足够

---

### 步骤 8：(可选) 真机微调

**脚本**: `training/finetune_xvf3800.py`
**适用场景**: 部署后发现某些环境/声音识别不佳，需要增量改进

```powershell
# 1. 将新采集的数据放到 custom_dataset/ 对应目录
# 2. 重新运行分割脚本
python training/split_dataset.py
# 3. 运行微调
conda activate wakefusion
python training/finetune_xvf3800.py
```

> 微调会在已有模型基础上继续训练，同样使用 EarlyStopping 防止过拟合。
> 微调完成后会覆盖 `xiaokang_xvf3800_pro.nemo`。

## ❓ 常见问题

### Q: 训练时报错 `_pickle.PicklingError` 或 `WinError 6`
**A**: 确保所有脚本中 `num_workers=0`。当前版本已全部设为 0。

### Q: 训练时报错 `correct_counts_k` 维度冲突
**A**: 当前版本已通过猴子补丁解决。如果仍然报错，请确认使用的是最新版 `fast_train.py`。

### Q: 训练时报错 GBK 编码错误
**A**: 当前版本的 `split_dataset.py` 使用 `ensure_ascii=True` 生成清单，已解决此问题。

### Q: EarlyStopping 在第 1 轮就停止了
**A**: 说明验证损失没有任何改善。检查数据是否正确生成，正负样本是否都存在。

### Q: 唤醒率很低，只有录音的人能唤醒
**A**: 需要更多不同人的真人录音 + TTS 多声音正样本。这是最关键的改进方向。

### Q: 训练过程会影响视觉/WebSocket 服务吗？
**A**: 不会。训练脚本与视觉服务、WebSocket 服务完全解耦，可以同时运行。
但训练完成后覆盖 `.nemo` 文件时，请先停止 `audio_service.py`。

## 🔧 参数调优建议

| 参数 | 默认值 | 调优建议 |
|------|--------|---------|
| `THRESHOLD` (audio_service.py) | 0.90 | 误唤醒多 → 升高到 0.95；漏唤醒多 → 降低到 0.85 |
| `MAX_EPOCHS` (fast_train.py) | 60 | 一般不需要修改，EarlyStopping 会自动控制 |
| `PATIENCE` (fast_train.py) | 10 | 数据量大可以增到 15；数据量小可以减到 5 |
| `BATCH_SIZE` (fast_train.py) | 32 | 显存不足时减到 16 |
| `BUFFER_DURATION` (audio_service.py) | 2.0 | 必须与训练数据时长一致，不要随意修改 |
